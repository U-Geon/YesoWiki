## 변경 사항 요약

`feat/005-premium-ui`에서 도입된 프리미엄 UI의 두 가지 버그를 수정합니다.

---

### 🐛 Fix 1: `.btn-ghost` 취소 버튼 패딩 미적용

**원인**: `globals.css`의 `.btn-ghost` 클래스에 `padding` 속성이 누락되어 있었음.  
`.btn-primary`는 `padding: 0.45rem 1rem`이 적용되어 있었으나 `.btn-ghost`에는 없었음.

**수정**: `.btn-ghost`에 동일한 `padding: 0.45rem 1rem` 추가

```diff
 .btn-ghost {
   display: inline-flex;
   align-items: center;
   ...
+  padding: 0.45rem 1rem;
   border-radius: 8px;
   cursor: pointer;
 }
```

---

### 🐛 Fix 2: 라이트모드 전환 시 반짝임 현상 (FOUC) 방지

**원인**: SSR 시점에 `<html data-theme="dark">`로 고정 렌더링되고, 클라이언트 `useEffect`에서 `localStorage` 값을 읽어 테마를 교체하기까지 시간 차가 발생함. 매 페이지 이동마다 서버 렌더링이 먼저 다크 테마로 페인트되어 반짝임 발생.

**수정**: `layout.tsx`의 `<head>` 내에 `strategy="beforeInteractive"` 인라인 스크립트 삽입. 브라우저가 HTML 파싱을 시작하자마자 `localStorage`에서 테마를 읽어 `data-theme`을 즉시 설정.

```diff
 <html lang="ko" suppressHydrationWarning>
+  <head>
+    <Script id="theme-init" strategy="beforeInteractive">{`
+      (function() {
+        try {
+          var theme = localStorage.getItem('theme') || 'dark';
+          document.documentElement.setAttribute('data-theme', theme);
+        } catch(e) {
+          document.documentElement.setAttribute('data-theme', 'dark');
+        }
+      })();
+    `}</Script>
+  </head>
   <body>...</body>
 </html>
```

- `try-catch` 처리: 프라이빗 브라우징 등 `localStorage` 접근 불가 환경에서 `dark` 폴백
- `html` 태그의 하드코딩된 `data-theme="dark"` 제거 → 스크립트가 항상 정확한 값으로 설정
